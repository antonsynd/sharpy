# Fix Task: R-0.1.3.4 - Fix Integer Division to Return float64

## Summary

**Issue**: Integer division (`/`) currently produces integer result instead of float64 per spec.

**Spec Requirement** (from `docs/language_specification/arithmetic_operators.md`):
> Integer types only | `float64` | Always promotes to `float64`

**Current Behavior**: `5 / 2` → `2` (int)
**Expected Behavior**: `5 / 2` → `2.5` (float64)

## Evidence

### Test Verification
The spec deviation tests in `src/Sharpy.Compiler.Tests/Integration/SpecDeviations/DivisionDeviationTests.cs` confirm the issue:

```
Generated C# for `5 / 2`:
    var result = x / y;  // where x: int, y: int
Output: 2 (integer truncation)
```

### Code Analysis
1. **`OperatorValidator.cs`** - Division uses same `InferNumericResultType()` as other arithmetic operators (no special handling for `/`)
2. **`RoslynEmitter.cs`** - Division maps directly to `SyntaxKind.DivideExpression` (C# native division)
3. **Conflicting tests**:
   - `OperatorValidatorTests.cs:177` - `ValidateBinaryOp_IntDivideInt_ReturnsInt()` expects `int` (WRONG)
   - `TypeCheckerTests.cs:318` - `ChecksDivisionProducesDouble()` expects `double` (CORRECT)

## Files to Modify

| File | Change Required |
|------|-----------------|
| `src/Sharpy.Compiler/Semantic/OperatorValidator.cs` | Add special case for `BinaryOperator.Divide` to return `float64` for integer operands |
| `src/Sharpy.Compiler/CodeGen/RoslynEmitter.cs` | Add lowering for integer division to cast operands to `double` |
| `src/Sharpy.Compiler.Tests/Semantic/OperatorValidatorTests.cs` | Fix test to expect `float64` instead of `int` |

## Implementation Steps

1. **OperatorValidator.cs** - In `TryResolveBuiltinOperator()`, when operator is `BinaryOperator.Divide` and both operands are integer types, return `float64` as result type

2. **RoslynEmitter.cs** - When generating division expression for integer operands:
   ```csharp
   // Instead of: x / y
   // Generate: (double)x / y
   ```

3. **Update/Fix tests** - Ensure `ValidateBinaryOp_IntDivideInt_ReturnsInt` expects `float64`

4. **Enable skipped tests** - Remove `Skip` attribute from tests in `DivisionDeviationTests.cs`

## Potential Breaking Changes

1. Code relying on `int / int = int` will need to use `//` for floor division
2. Variables inferred from division will change type (e.g., `x = 10 / 2` becomes `float64`)
3. Augmented assignment `x: int; x /= 2` will need special handling or error
