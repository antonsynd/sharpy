# Implementation Plan: R-0.1.3.3 - Add Spec Deviation Tests for Division

## Executive Summary

**Current State**: Division of two integers in Sharpy currently produces an integer result (C# semantics), not a float64 as specified.

**Evidence Found**:
1. `OperatorValidatorTests.cs:177` - Test named `ValidateBinaryOp_IntDivideInt_ReturnsInt()` expects `int / int = int` (WRONG per spec)
2. `TypeCheckerTests.cs:318` - Test named `ChecksDivisionProducesDouble()` expects `int / int = double` (CORRECT per spec)
3. `RoslynEmitter.cs:1953` - Division maps directly to `SyntaxKind.DivideExpression` (C# native division)
4. `OperatorValidator.cs:583` - Division uses same `InferNumericResultType()` as other operators (no special handling)

**Spec Requirement** (from `docs/language_specification/arithmetic_operators.md`):
> Integer types only | `float64` | Always promotes to `float64`

## Potential Risks and Questions

### Risks

1. **Breaking Change**: Existing code that relies on `int / int = int` will break
   - Mitigation: This is intentional - the spec requires Python semantics

2. **Test Failures**: The existing test `ValidateBinaryOp_IntDivideInt_ReturnsInt` will fail
   - Mitigation: This test is incorrect per spec and should be fixed

3. **Type Inference Cascading**: Variables inferred from division will change type
   - Example: `x = 10 / 2` currently infers `x: int`, will become `x: float`
   - This may cause downstream type errors in user code

4. **Augmented Assignment**: `x /= 2` where `x: int` will need special handling
   - May need to reject `int /= int` or auto-convert

### Questions

1. **Q**: Should `x: int = 5 / 2` be allowed with implicit truncation, or error?
   - **Recommendation**: Error - require explicit `int(5 / 2)` or `5 // 2`

2. **Q**: What about augmented assignment `x: int = 10; x /= 2`?
   - **Recommendation**: Error - use `x //= 2` for integer division

3. **Q**: Should there be a warning when division result is assigned to int?
   - **Recommendation**: No warning, just error - be strict per spec
