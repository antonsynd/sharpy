# Sharpy v0.5 Feature List

This document provides a comprehensive, categorized list of all features that must be implemented and validated for Sharpy v0.5. Each feature is described at a granular level to enable systematic verification across the entire toolchain (lexing, parsing, semantic analysis, code generation, and testing).

## 1. Lexical Structure

### 1.1 Source File Encoding
- **Feature**: UTF-8 encoding support
- **Feature**: LF (`\n`) line ending support
- **Feature**: CRLF (`\r\n`) line ending support
- **Feature**: Optional BOM handling (accept but not required)

### 1.2 Indentation
- **Feature**: Exactly 4 spaces per indentation level (enforced)
- **Feature**: Tabs rejected for indentation
- **Feature**: Mixed spaces and tabs cause lexical error
- **Feature**: Indentation increase generates INDENT token
- **Feature**: Indentation decrease generates DEDENT token(s)
- **Feature**: Indentation must decrease to previous level (not arbitrary)
- **Feature**: End-of-file generates DEDENT for each remaining level

### 1.3 Comments
- **Feature**: Single-line comments starting with `#`
- **Feature**: Comments extend to end of line
- **Feature**: `#` inside string literals is not a comment
- **Feature**: Docstrings (triple-quoted strings as first statement in module/class/function)

### 1.4 Identifiers
- **Feature**: Identifiers start with letter or underscore
- **Feature**: Identifiers contain letters, digits, underscores
- **Feature**: Case-sensitive identifiers
- **Feature**: Keywords cannot be used as identifiers
- **Feature**: Unicode letters allowed (but discouraged)

### 1.5 Whitespace
- **Feature**: Spaces, tabs, form feeds recognized as whitespace
- **Feature**: Leading whitespace (indentation) is significant
- **Feature**: Trailing whitespace ignored
- **Feature**: Blank lines ignored

### 1.6 Line Continuation
- **Feature**: Explicit continuation with backslash `\`
- **Feature**: Implicit continuation inside `()`, `[]`, `{}`
- **Feature**: Cannot continue in middle of identifier
- **Feature**: Cannot continue inside single-line strings

### 1.7 Literal Names (Backticks)
- **Feature**: Backtick-quoted identifiers prevent name transformation
- **Feature**: Backticks allow using keywords as identifiers
- **Feature**: Backticks preserve exact casing
