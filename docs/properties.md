About property declaration...

Properties can have just a getter or just a setter. If absent, it means
it doesn't exist (i.e. readeable only or writeable only).

For access modifiers, I think maybe it's better to split the property declaration
into separate parts:

```python
get value(self) -> int:
    return self.__value

# Note, the type annotation here is optional because it is already inferred
# from the getter. It would be the same in the opposite direction if the
# setter was defined first with a type annotation and the getter does not need
# to specify the return type
set value(self, v: int):
    self.__value = v
```

And then one could clearly make the access modifiers like so:

```python
get value(self) -> int:
    # public
    return self.__value

# protected access
set _value(self, v):
    self.__value = v
```

If one only wants to specify the access modifier, then:

```python
get value -> int: auto
set _value: auto
```

With the new `auto` (soft) keyword indicating the default implementation. This
is akin to the `auto()` function from the enum module in Python.

It is also possible to auto the entire property (with the same access modifier
on both the getter and setter) using the soft keyword `property`

```python
# This property has automatically generated getter/setters with file-only
# visibility with default initialized value for the type (in this case, it is 0)
property $$value: int

# This property is protected with auto-generated getter/setters, with initialized
# value of 5
property _other_value: int = 5
```

So in the end, there is no need for ellipsis for auto-generated implementations.
Ellipsis is still for "abstract" methods from protocols.

```python
# Auto-generated getter/setter with same access modifier, $ (= internal)
# and initialized value expression. the backing field is generated by the
# compiler as private access and its name is mangled. Corresponds to
# C#:
#    internal int Value { get; set; } = 5;
property $value: int = 5

# Read-only public auto-property with no user-defined initial value (defaults to
# type's initial value. Has backing field.
get property length: int

# Write-only private auto-property with initial value. Has backing field.
set property _size: int = 0

# Explicitly provided getter/setters (the absence of one or the other means
# read-only/write-only respectively). No backing field is auto-generated.
# Type annotation is required (for now) on both getter and setter if both
# are defined. Getter/setter is inferred from the argument types. Getter
# only has self, setter has self plus one other argument.
property _dimensions(self) -> int:
    return self.__some_backing_field

property __dimensions(self, v: int):
    self.__some_backing_field = v

# In protocols (or i guess abstract classes), an abstract property can
# be defined

# No initial value can be defined. Providing get/set in front of property
# will mean the protocol only requires the getter/setter.
property num_characters: int
get property num_words: int
set property num_sentences: int

# An abstract explicit property's getter/setters can also be defined using
# ellipsis.
property num_tokens(self) -> int: ...
property num_tokens(self, v: int): ...
```
