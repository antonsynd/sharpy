# Sharpy Compiler Health Audit — 2026-05-11

## Executive Summary

The Sharpy compiler is in **excellent health** across all seven audit dimensions. With 13,333 tests (13,332 passing, 1 flaky, 0 skipped), 1,817 file-based test fixtures, and 100% TODO/issue compliance, the codebase demonstrates strong engineering discipline. Architecture is clean with zero circular dependencies between subsystems, and all major classes are well-partitioned into partial files under 1,800 lines each.

The most significant finding is a **flaky timing-dependent test** (`ModuleDiscoveryWorkflowTests.Workflow_CachingWorks_FasterOnSecondLoad`) that fails intermittently in full suite runs but passes in isolation. The only structural concern is that parallel compilation infrastructure (Issue #610) remains partially implemented — the design is sound but the merge phase and `GlobalSymbolTable` implementation are pending.

The LSP server has exceptional coverage with 22 handler types, 51 test files, 8 refactoring providers, and proper thread safety via `ConcurrentDictionary`, `SemaphoreSlim`, and `Interlocked` operations. .NET compliance is clean: zero C# 9.0 violations in Sharpy.Core, correct multi-targeting, and complete type mappings.

## Critical Findings

### 1. Flaky Test: `Workflow_CachingWorks_FasterOnSecondLoad`
- **Location**: `src/Sharpy.Compiler.Tests/Integration/ModuleDiscoveryWorkflowTests.cs`
- **Behavior**: Fails intermittently in full suite runs (timing-dependent), passes in isolation
- **Impact**: CI reliability
- **Recommendation**: Convert to non-timing assertion (e.g., verify cache hit rather than measuring elapsed time)

## Warnings

### 1. Unused Diagnostic Codes (19 of 177)
Nineteen diagnostic codes are defined in `DiagnosticCodes.cs` but never emitted. All are intentionally reserved or superseded, but two should be formally deprecated:
- `SPY0372` (DelegateWithBody) — never implemented
- `SPY0457` (UnsupportedDunderReversed) — `__reversed__` now supported, code is obsolete

### 2. Parallel Compilation Partially Implemented
Design document exists (`docs/design/parallel-compilation.md`, Issue #610) and thread-safety annotations are in place, but:
- `SymbolTable` uses non-concurrent `Dictionary<string, Scope>` and `Stack<Scope>`
- `GlobalSymbolTable` merge logic not yet implemented
- `ProjectCompiler` phase split not yet wired up

### 3. RoslynEmitter Growth Trajectory
Currently 16,927 lines across 22 partial files. The largest partials are approaching 1,500 lines:
- `RoslynEmitter.TypeDeclarations.cs` — 1,435 lines
- `RoslynEmitter.Expressions.Access.cs` — 1,301 lines

Monitor and split if files exceed 1,800 lines.
