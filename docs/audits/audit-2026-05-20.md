# Sharpy Compiler Health Audit — 2026-05-20

## Executive Summary

The Sharpy compiler is in excellent health. **All 14,574 tests pass** (0 failures, 3 skipped) across the compiler, stdlib, and LSP server. The architecture is clean with **zero circular dependencies** between Parser, Semantic, and CodeGen layers. TODO/FIXME compliance is at **100%** — all 6 deferred work items reference GitHub issues.

The codebase has grown to ~98K compiler LOC, ~27K stdlib LOC, and ~12K LSP LOC, supported by 1,958 file-based test fixtures and 128 property-based test files. The LSP server implements 30+ handlers with 53 test files and 4 E2E test suites. Future readiness is strong: parallel builds, REPL, and debugger source mapping are all architecturally ready today.

Key areas for attention: the CLI's `Program.cs` (2,188 LOC) is monolithic, 10 formatting test fixtures are skipped pending formatter implementation, and 12 diagnostic codes are allocated but not yet emitted. None of these are blocking.

## Critical Findings

**None.** No critical issues found in any audit dimension.

## Warnings

1. **CLI Program.cs is monolithic** (2,188 LOC) — Should be refactored into per-command handler files (~15 files, 150 LOC each). High churn risk as CLI features grow.

2. **10 skipped formatting test fixtures** — All in `TestFixtures/Formatting/`. Blocked on formatter implementation (trivia roundtrip not yet supported). Tracked and expected.

3. **3 skipped programmatic tests** — `SourceGeneratorTests.GeneratorClass_Compiles_WithoutErrors` and 2 stress tests (`HighFuelPrograms`, `DeepNestingPrograms`). Should verify these are intentionally skipped with documented reasons.

4. **12 allocated diagnostic codes** — Defined in `DiagnosticCodes.cs` with `// Allocated` status but not yet emitted. Expected for planned features but should be tracked.

## Opportunities

1. **Extract collection type tags to constants** — Some CodeGen files use string literals `"List"`, `"Set"`, `"Dict"` that could use `CSharpTypeNames` constants for consistency.

2. **Document SPY0000** — The generic error code used by `StructuredLogger` should be explicitly documented in `DiagnosticCodes.cs`.

3. **Emitter interface extraction** — Currently tightly coupled to Roslyn's SyntaxFactory. An `ICodeEmitter` interface would enable alternative backends (LLVM, interpreter) if ever needed. Large effort (~150h), low urgency.
