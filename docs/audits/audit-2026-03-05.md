# Sharpy Compiler Health Audit — 2026-03-05

## Executive Summary

The Sharpy compiler is in **excellent health**. All 7,807 tests pass (99.99% pass rate, 1 documented skip), .NET compliance is fully verified with zero issues, and the codebase maintains strong architectural discipline across 82,308 lines of C# and 1,161 integration test fixtures.

The most significant finding is a **single circular dependency** between Semantic and Project layers (via `DependencyGraphBuilder`). Beyond that, issues are limited to stale documentation (partial file counts), low warning test coverage (5.8%), and opportunities for further partial file splitting as the largest files approach 2,000 lines.

**Top 3 priorities**: (1) Fix Semantic ↔ Project circular dependency, (2) Update CLAUDE.md with correct partial file counts, (3) Expand warning and snapshot test coverage for underserved categories.

## Critical Findings

### CIRC-001: Semantic ↔ Project Circular Dependency
`ImportResolver.cs` imports from `Sharpy.Compiler.Project` (uses `DependencyGraphBuilder`), while `ProjectCompiler.Phases.cs` imports `Sharpy.Compiler.Semantic`. This is a bidirectional dependency violating the intended layering.

**Resolution**: Move `DependencyGraphBuilder`/`IDependencyGraphBuilder` to Semantic (import tracking is a semantic concern), or extract a shared interface both layers depend on.

## Warnings

### WARN-001: Stale Documentation for Partial File Counts
- CLAUDE.md states RoslynEmitter has 12 partials → **actual: 16**
- CLAUDE.md states TypeChecker has 8 partials → **actual: 9**
- CodeGen/README.md says "11 files total" → **actual: 16**

### WARN-002: Stale `using` in Parser Layer
`Parser/Parser.Types.cs:7` has `using Sharpy.Compiler.Semantic;` but no Semantic types are referenced. Latent layering violation.

### WARN-003: Validation Logic in TypeChecker.Utilities
`TypeChecker.Utilities.cs` contains `ValidateConstructorOverloads`, `ValidateStructRules`, `ValidateEnumRules` (lines 425–897) that may be self-contained analyses better suited for ValidationPipeline.

### WARN-004: Position-Based Diagnostic Deduplication
TypeChecker and OperatorValidator deduplicate errors by `(line, column)` matching. A correlation ID would be more robust.

### WARN-005: Empty Test Fixture Categories
`cross_module_inheritance/`, `imports/`, `module_imports/`, `multi_file/` directories exist but contain 0 fixtures.

### WARN-006: Sparse Codegen Snapshot Coverage
55 snapshots cover only 4.7% of 1,161 fixtures. Categories with 0 snapshots: async, decorators, enums, exceptions, functions, generators, lambdas, modules, operators, statements.
