#### 1. Division Result Type Not Fully Spec-Compliant
- [ ] **Section: Arithmetic Operators - Division `/`**
  - **Spec says:** "Integer types only → Result is `float64`" - Division of two integers must always return float64.
  - **Implementation:** `OperatorValidator.InferNumericResultType()` at line 811 delegates to `PrimitiveCatalog.GetPromotedType()`, which returns the promoted numeric type (int+int→int) rather than forcing float64 for division.
  - **Impact:** Medium - The division operator doesn't force float64 result for integer operands.

#### 2. `__floordiv__` and `__pow__` Dunders Incorrectly Supported
- [ ] **Section: Dunder Methods - Unsupported Dunders**
  - **Spec says:** "`__pow__` Not supported - `**` is not an overloadable operator in C#" and "`__floordiv__` Not supported - Use `__div__` for `/` operator; `//` handled specially"
  - **Implementation:** `OperatorValidator.BinaryOperatorToDunder()` at lines 152-154 maps `FloorDivide` → `__floordiv__` and `Power` → `__pow__`
  - **Impact:** Medium - Allows user-defined `__floordiv__` and `__pow__` which cannot be emitted to C#.

#### 3. In-place Operators Partially Supported
- [ ] **Section: Assignment Operators**
  - **Spec says:** "In the current version of Sharpy, user definitions of assignment operators like `+=` via dunder methods (e.g. `__iadd__`) are not supported."
  - **Implementation:** `OperatorValidator.AssignmentOperatorToInPlaceDunder()` at lines 894-912 maps all in-place dunders (`__iadd__`, `__isub__`, etc.) and `ValidateAugmentedAssignment` attempts to find them.
  - **Impact:** Low - Implementation attempts to use `__iadd__` etc. but falls back to binary operators. Spec says these shouldn't be user-definable.

#### 4. Type Casting with `to` Operator Not Validated
- [ ] **Section: Type Casting**
  - **Spec says:** "The `to` operator performs type casting... The compiler rejects casts that are statically known to be impossible"
  - **Implementation:** `TypeChecker.CheckTypeCast()` at line 1629-1633 only resolves the target type without validating cast feasibility.
  - **Impact:** Medium - Invalid casts like `int to str` are not rejected at compile time.

#### 5. `__truediv__` vs `__div__` Naming
- [ ] **Section: Dunder Methods - Arithmetic Operators**
  - **Spec says:** "`__truediv__` is renamed to `__div__` to reflect the lack of a contrasting `__floordiv__`."
  - **Implementation:** `OperatorValidator.BinaryOperatorToDunder()` at line 151 maps `Divide` → `__truediv__` (Python naming).
  - **Impact:** Low - Uses Python naming (`__truediv__`) instead of spec-mandated Sharpy naming (`__div__`).

#### 6. String Repetition Not Supported
- [ ] **Section: String Operators**
  - **Spec says:** "`*` Repetition | `'ab' * 3` | `'ababab'`" and "`3 * 'ab'` | `'ababab'`"
  - **Implementation:** `OperatorValidator.TryResolveBuiltinOperator()` only handles str+str concatenation (line 600-609), not str*int or int*str repetition.
  - **Impact:** Low - String repetition with `*` operator is not type-checked.

#### 7. Self Parameter Validation Incomplete for Static Methods
- [ ] **Section: Dunder Methods**
  - **Spec says:** Static methods shouldn't require self.
  - **Implementation:** `TypeChecker.CheckFunction()` at lines 195-209 validates self for all methods without checking `@staticmethod` decorator first.
  - **Impact:** Low - Static methods are incorrectly required to have self parameter.

### Recommendations

1. **Division Result Type**: Modify `OperatorValidator.TryResolveBuiltinOperator()` to special-case the `/` operator for integer operands, forcing the result to `float64` per spec.

2. **Remove `__floordiv__` and `__pow__` Support**: Update `BinaryOperatorToDunder()` to return `null` for these operators, as they cannot be user-overloaded per spec.

3. **Disable In-place Dunder Validation**: Update `AssignmentOperatorToInPlaceDunder()` to return `null` for all in-place dunders until C# 14 support, per spec.

4. **Add Type Cast Validation**: Enhance `CheckTypeCast()` to verify cast feasibility using inheritance relationships and primitive conversion rules.

5. **Rename `__truediv__` to `__div__`**: Update the dunder mapping to use Sharpy naming (`__div__`) instead of Python naming (`__truediv__`).

6. **Add String Repetition**: Add cases in `TryResolveBuiltinOperator()` for `str * int` and `int * str` returning `str`.

7. **Static Method Self Check**: Modify `CheckFunction()` to skip self validation when `@staticmethod` decorator is present.
