# Task 0.1.9.9: Implement Generic Type Parsing

## Summary

Parse generic class/method definitions with `[T]` syntax. Most infrastructure already exists for classes - the main work is adding generic type parameter support to **functions/methods**.

## Current State Analysis

### Already Implemented
- **Lexer**: `LeftBracket`/`RightBracket` tokens with bracket depth tracking
- **Parser**: Full support for class/struct/interface generic parameters `[T, U]` (lines 372-385 in Parser.cs)
- **AST**: `ClassDef`, `StructDef`, `InterfaceDef` all have `TypeParameters: List<string>`
- **Semantic**: `TypeSymbol` has `TypeParameters` field, `GenericType` is fully modeled
- **Tests**: Comprehensive tests for class generics exist

### Missing for Method Generics
1. **AST**: `FunctionDef` lacks `TypeParameters` field
2. **Parser**: `ParseFunctionDef()` doesn't parse `[T, U]` between name and `(`
3. **Semantic**: `FunctionSymbol` lacks `TypeParameters` field

```csharp
Expect(TokenType.Def);
var name = ExpectIdentifier();

// Parse type parameters [T, U] - same pattern as ParseClassDef
var typeParams = new List<string>();
if (Current.Type == TokenType.LeftBracket)
{
    Advance();
    do
    {
        typeParams.Add(ExpectIdentifier());
        if (Current.Type == TokenType.Comma)
            Advance();
        else
            break;
    } while (true);
    Expect(TokenType.RightBracket);
}

Expect(TokenType.LeftParen);
```

Also update the return statement to include `TypeParameters = typeParams`.

### Step 4: Add Parser Tests for Generic Functions

**File**: `src/Sharpy.Compiler.Tests/Parser/ParserTests.cs`

Add new tests in the generics region:

```csharp
[Fact]
public void ParseGenericFunctionDefinition()
{
    var source = @"
def identity[T](value: T) -> T:
    return value
";
    var module = Parse(source);
    var funcDef = module.Body[0].Should().BeOfType<FunctionDef>().Subject;
    funcDef.Name.Should().Be("identity");
    funcDef.TypeParameters.Should().HaveCount(1);
    funcDef.TypeParameters[0].Should().Be("T");
    funcDef.Parameters.Should().HaveCount(1);
    funcDef.Parameters[0].Type!.Name.Should().Be("T");
    funcDef.ReturnType!.Name.Should().Be("T");
}

[Fact]
public void ParseGenericFunctionWithMultipleTypeParameters()
{
    var source = @"
def pair[T, U](first: T, second: U) -> tuple[T, U]:
    return (first, second)
";
    var module = Parse(source);
    var funcDef = module.Body[0].Should().BeOfType<FunctionDef>().Subject;
    funcDef.TypeParameters.Should().HaveCount(2);
    funcDef.TypeParameters[0].Should().Be("T");
    funcDef.TypeParameters[1].Should().Be("U");
}

[Fact]
public void ParseGenericMethodInClass()
{
    var source = @"
class Utils:
    def find_max[T](self, a: T, b: T) -> T:
        pass
";
    var module = Parse(source);
    var classDef = module.Body[0].Should().BeOfType<ClassDef>().Subject;
    var methodDef = classDef.Body[0].Should().BeOfType<FunctionDef>().Subject;
    methodDef.TypeParameters.Should().HaveCount(1);
    methodDef.TypeParameters[0].Should().Be("T");
}

[Fact]
public void ParseFunctionWithoutTypeParameters()
{
    // Ensure non-generic functions still work
    var source = @"
def add(a: int, b: int) -> int:
    return a + b
";
    var module = Parse(source);
    var funcDef = module.Body[0].Should().BeOfType<FunctionDef>().Subject;
    funcDef.TypeParameters.Should().BeEmpty();
}
```

## Potential Risks

1. **Disambiguation**: Square brackets are overloaded:
   - Type parameters: `def foo[T](...)`
   - Type arguments in annotations: `list[int]`
   - Indexing: `arr[0]`

   Context-based disambiguation already works for classes - the parser expects `[` immediately after the function name, before `(`.

2. **Trailing comma**: Consider whether `[T,]` should be allowed. Current class parsing doesn't allow it - keep consistent.

3. **Nested generics in type annotations**: Already supported via `ParseTypeAnnotation()`.

## Questions for Clarification

1. Should we support type parameter constraints like `[T: Comparable]`? (Probably future task)
2. Should we support default type arguments like `[T = int]`? (Probably future task)
