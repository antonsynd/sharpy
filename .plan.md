## Current Implementation Status

### Single-Line Comments: ✅ IMPLEMENTED

**Location**: `src/Sharpy.Compiler/Lexer/Lexer.cs`

The lexer handles single-line comments through the `ReadComment()` method (lines 488-507):

```csharp
private Token ReadComment()
{
    // Skip the '#'
    _position++;
    _column++;

    while (_position < _source.Length && _source[_position] != '\n' && _source[_position] != '\r')
    {
        _position++;
        _column++;
    }

    // Comments are skipped - return next token
    return NextToken();
}
```

**Key behaviors**:
- Comments starting with `#` are recognized at line 259-260 in the main tokenization switch
- Comments are **skipped entirely** (no Comment token is emitted)
- Comment content is discarded, and `NextToken()` is called recursively
- Handles comments at end of line, on own line, indented, and inside brackets

### Triple-Quoted Strings: ✅ IMPLEMENTED

**Location**: `src/Sharpy.Compiler/Lexer/Lexer.cs` (lines 567-618)

- `ReadTripleQuotedString()` handles both `"""..."""` and `'''...'''`
- Multi-line content with proper newline tracking
- These are tokenized as `String` tokens (not special docstring tokens)
- Parser/semantic layer can later interpret triple-quoted strings at statement position as docstrings

### Design Questions (Already Decided)

1. **Q**: Should comments produce tokens for doc tools?
   - **Current Decision**: No - comments are skipped entirely
   - **Note**: `TokenType.Comment` exists but is intentionally not emitted
   - **Rationale**: Simplifies token stream; doc tools could re-parse if needed

2. **Q**: How are docstrings handled?
   - **Current Decision**: Triple-quoted strings are tokenized as `String`
   - **Rationale**: Parser/semantic layer determines if they're docstrings based on position

### Known Behaviors (Working as Designed)

- Comments are **completely invisible** in the token stream
- Triple-quoted strings are regular `String` tokens (not special docstring tokens)
- Indented comments don't produce spurious NEWLINE/INDENT tokens
- Comments inside brackets `()[]{}` are properly skipped
