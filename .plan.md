Based on task description, verify these are NOT currently tested:
1. **Uppercase prefix variants** - `0X`, `0B`, `0O` (uppercase) may not have explicit tests
2. **Scientific notation `E`** - uppercase `E` may not have explicit tests

## Potential Risks & Questions

### Risks
1. **Test coverage gaps** - Some case variants may lack explicit tests
2. **Edge cases** - Combinations like `0xFFUL` (hex with suffix) may not be tested

### Questions for Clarification
1. Should hex/binary/octal support type suffixes? (e.g., `0xFFL`)
   - Current implementation: NO - these return immediately without suffix parsing
2. Are there any C# numeric literal features we're intentionally NOT supporting?
