Per `type_casting.md:96-100`:
> | Narrowing (e.g., `int64` → `int32`) | Throws/None on overflow |

**However**, this is a semantic/runtime behavior concern rather than a code generation bug - the generated C# is syntactically correct. The overflow-checking behavior would need to be:
1. Added to the project-level `<CheckForOverflowUnderflow>true</CheckForOverflowUnderflow>`, OR
2. Wrapped in `checked()` at the expression level for numeric coercions
