# Implementation Plan: R-0.1.2.2 - Audit Unity C# 9.0 Compatibility (File-Scoped Namespaces)

## Summary

The task is to address a C# 10+ feature (`FileScopedNamespaceDeclaration`) used in `RoslynEmitter.cs` that violates the project's C# 9.0 compatibility requirement for Unity support.

**Current code (line 92):**
```csharp
var namespaceDecl = FileScopedNamespaceDeclaration(namespaceName)
    .WithMembers(SingletonList<MemberDeclarationSyntax>(moduleClass));
```

**Problem:** `FileScopedNamespaceDeclaration` generates `namespace Foo;` syntax which is C# 10+. Unity uses C# 9.0, so this violates **Axiom 1** of the language specification.

## Analysis

### Project Requirements Confirm C# 9.0 Target

From `docs/language_specification/introduction.md`:
> **Axiom 1 — .NET Runtime:** Sharpy compiles to C# 9.0 and executes on the .NET CLR.

From `.github/agents/net-axiom-guardian.agent.md`:
> C# 10+ ❌ Not Allowed
> ```csharp
> namespace MyNamespace;                       // File-scoped namespace
> ```

### Distinction: Compiler vs Generated Code

**Important distinction:**
- The **Sharpy compiler itself** uses `LangVersion=latest` and targets `net10.0` — this is fine
- The **generated C# code** must be C# 9.0 compatible for Unity consumers

The issue is that `RoslynEmitter` generates C# code that uses file-scoped namespaces (`namespace Foo;`), which won't compile in Unity's C# 9.0 environment.

### Current Generated Output

```csharp
// Generated by Sharpy (C# 10+ syntax - WRONG)
using System;
using Sharpy.Core;

namespace MyModule;  // <-- File-scoped namespace (C# 10+)

public static class Module
{
    // ...
}
```

### Required Generated Output

```csharp
// Generated by Sharpy (C# 9.0 compatible - CORRECT)
using System;
using Sharpy.Core;

namespace MyModule  // <-- Block-scoped namespace (C# 9.0)
{
    public static class Module
    {
        // ...
    }
}
```

## Implementation Approach

### Step 1: Replace FileScopedNamespaceDeclaration with NamespaceDeclaration

**File:** `src/Sharpy.Compiler/CodeGen/RoslynEmitter.cs` (line 92)

**Change from:**
```csharp
var namespaceDecl = FileScopedNamespaceDeclaration(namespaceName)
    .WithMembers(SingletonList<MemberDeclarationSyntax>(moduleClass));
```

**Change to:**
```csharp
var namespaceDecl = NamespaceDeclaration(namespaceName)
    .WithMembers(SingletonList<MemberDeclarationSyntax>(moduleClass));
```

This single change will generate block-scoped namespaces instead of file-scoped ones.

### Step 2: Update Tests to Expect Block-Scoped Namespaces

**File:** `src/Sharpy.Compiler.Tests/CodeGen/RoslynEmitterModuleTests.cs`

Existing tests check for `namespace X` in output. Most should still pass since both formats contain `namespace X`. However, we should:

1. Verify tests still pass (likely no changes needed)
2. Optionally add an explicit test verifying block-scoped syntax:

```csharp
[Fact]
public void GenerateCompilationUnit_GeneratesBlockScopedNamespace_ForCSharp9Compatibility()
{
    // Arrange
    var module = new Module { Body = new List<Statement>() };
    var context = new CodeGenContext { SourceFilePath = "test.spy" };
    var emitter = new RoslynEmitter(context);

    // Act
    var result = emitter.GenerateCompilationUnit(module);
    var code = result.NormalizeWhitespace().ToFullString();

    // Assert - Should use block syntax, not file-scoped
    Assert.Contains("namespace Test\n{", code.Replace("\r\n", "\n"));
    Assert.DoesNotContain("namespace Test;", code);
}
```

### Step 3: Verify Generated Code Compiles with C# 9.0

Run the build with explicit C# 9.0 language version to verify compatibility:

```bash
# Generate some Sharpy code and compile with C# 9.0
dotnet build /p:LangVersion=9.0 samples/SampleModule/SampleModule.csproj
```

## Potential Risks

### Low Risk: Formatting Differences

Block-scoped namespaces add one level of indentation to all members. This is a cosmetic difference that:
- May cause minor test assertion changes if tests check exact formatting
- Is functionally equivalent

### Low Risk: Generated Code Size

Block-scoped namespaces add:
- Opening brace `{`
- Closing brace `}`
- One additional indentation level

This is negligible.

## Questions for Clarification

### Question 1: Is Unity C# 9.0 compatibility still required?

**Evidence supporting "Yes":**
- Language spec explicitly states C# 9.0 target
- Multiple agent files reference Unity compatibility
- `.github/agents/unity-compatibility-guardian.agent.md` exists

**Recommendation:** Proceed with the fix since all documentation confirms C# 9.0 is required.

### Question 2: Should generated code style be configurable?

Some users might prefer file-scoped namespaces for non-Unity targets. A future enhancement could add:
```csharp
// In ProjectConfig or CodeGenContext
public bool UseFileScopedNamespaces { get; init; } = false;
```

**Recommendation:** Out of scope for this task. The default should be C# 9.0 compatible per Axiom 1.
